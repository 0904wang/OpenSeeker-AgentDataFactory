## Analysis

The batch/resume runner worked as intended. It refreshed exports every 50 completed samples, so intermediate summaries were available at 50, 150, 300, 450, 650, 800, 950, and 1,000 samples. This prevents the previous single-shot failure mode where one long-tail request delays all artifact export.

The teacher success rate was high: 997/1,000 samples came from DeepSeek teacher output and 3/1,000 used deterministic fallback after invalid or empty JSON responses. The verifier accepted all 1,000 samples because deterministic evidence, tool calls, and trajectories remained internally consistent.

This dataset is usable as a 1k SFT pilot and as evidence that the data factory can generate, filter, and export multiple training formats. It should not yet be treated as a final 20k/50k dataset because the seed bank is still small and repeated over 1,000 variants.
